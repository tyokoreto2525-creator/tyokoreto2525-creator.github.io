GrapheneOS完全ガイド

1. セキュリティ特化のAndroid

GrapheneOSとは？

あなたは今、どんなAndroidを使っているだろうか。ゲーミングに特化した機種、カメラ性能を重視した機種、動画視聴に最適化された機種……Androidにはさまざまなバリエーションがある。

しかし今回は「性能」や「カメラ」ではなく、セキュリティに特化したAndroid を紹介したい。軍事や企業向けのセキュリティ特化Androidも存在するが、非常に高価だ。しかし GrapheneOS（グラフェンOS）なら無料で導入でき、市販のGoogle Pixelを買えば誰でも利用可能である。

正確には「機種」ではなく「OS」による強化。そのOSの名前が GrapheneOS（グラフェンOS） だ。

GrapheneOSが動作するのはGoogle Pixelシリーズ。流れとしては、Google Pixelを購入 → GrapheneOSを導入 → セキュリティ強化されたPixelが完成という形になる。

GrapheneOSのメリットとデメリット

メリット

  • ゼロデイ攻撃に強い防御力
    メモリ安全性やカーネル保護が強化され、未知の攻撃にも比較的耐性がある。
  • アプリの動きを細部まで制御可能
    「このアプリはネット禁止」「センサーを使わせない」といった細かい設定ができ、情報漏洩を防げる。
  • コピー情報は自動で消去
    クリップボードの内容は一定時間で自動消去され、他アプリからの不正取得を防げる。
  • Googleサービスを選んで使える
    Google Playをサンドボックス化して実行できるため、必要に応じて利用可能。不要なら完全に外すこともできる。
  • オープンソースによる透明性
    ソースコードが公開され、研究者や専門家が自由に監査できる。
  • Pixelのハードをフルに活用
    Titan MチップやセキュアブートなどPixel独自の強力なセキュリティ機能をそのまま利用できる。
  • プロファイル分けで安全に使い分け
    「仕事用」「プライベート用」を完全に分離できる。

デメリット

  • Googleアシスタントが使えない（プライバシー保護のため非対応）。
  • Pixel特有の一部機能が制限（AIカメラ処理などGoogleサーバー依存部分）。
  • Google Playストアがそのまま使えない → 「Sandboxed Google Play」や「Aurora Store」を利用。
  • 一部アプリが動作しない（銀行アプリ・キャリア決済系アプリ）。
  • 利便性よりセキュリティ重視の設計 → 設定や管理に手間。
  • サポート外の可能性 → Pixel保証は残るがGoogleサポート対象外。

通常アプリは使えるのか？

GrapheneOSでもLINEやYouTubeなど多くのアプリは利用できる。

「Sandboxed Google Play」を導入すれば純正のGoogle Playストア・GMSをそのまま利用可能。Aurora Storeも代替手段になる。

  • Google Play → 自動更新あり
  • Aurora Store → 手動更新のみ

※LINEやTwitterなどサーバー保存型はそのまま利用可能。

⚠ 注意点

  • Googleサービス制限あり。Gmail/DriveはSandboxed経由で利用可能。
  • キャリアアプリ（au/doco/SoftBank）はほぼ非対応。
  • 一部ゲーム（Play Integrity必須）は動作不可。

おすすめ層：「キャリアアプリや便利機能を捨ててもいい」「連絡用途中心」「情報漏洩リスクを徹底的に下げたい」

Pixel購入時に注意すべき点

  • 対応状況：新機種すぐ対応とは限らない → 公式サイト確認必須。
  • サポート期限：古いモデルは安いがセキュリティ更新が切れると危険。
  • SIMカード対応：nanoSIM / eSIM対応を確認。

Pixelに戻せるのか？

戻せるが必ずデータは消える。ブートローダーを再ロックして純正Androidを再フラッシュすればOK。ただし文鎮化リスクあり。

まとめ

メリット：世界最高水準のセキュリティ＋プライバシー制御

デメリット：便利機能を犠牲にする

GrapheneOSは「便利さより安全」を選ぶ人のためのOS。 日常重視なら純正Pixel、最高セキュリティならGrapheneOS、と使い分けがベスト。

GrapheneOSの導入方法（概要）

必要なもの：

  • PC（Google Chrome/Chromium系推奨）
  • Pixel端末
  • 接続用ケーブル（データ通信対応）

Pixel側の準備

  • 開発者モードON → ビルド番号7回タップ
  • OEMロック解除ON
  • USBデバッグON

インストール手順

  1. PixelをFastbootモードにする
  2. PCで 公式Web Installer を開く
  3. 「Unlock bootloader → Download release → Flash release → Lock bootloader」を順に実行
  4. Fastbootで「Start」を選び起動 → 完了

⚠ 注意点

  • 作業中はデータ全消去
  • ケーブルを抜かない／電源切らない
  • 毎回の起動時に黄色警告は正常
  • 必ず「Lock bootloader」を最後に実行

2. GrapheneOS 導入後

プロファイル分け（メインとサブ）

GrapheneOSの強みはユーザープロファイル分離にあります。Windowsの「ユーザーアカウント」に近いイメージです。 ※なお、この挙動はAndroid標準の複数ユーザー機能による仕様で、GrapheneOS独自の制限ではありません。ただGrapheneOSはこの標準機能を強化されたセキュリティ基盤上で活用できるのが特徴です。

メインプロファイル（オーナー）

  • 電話番号・SMSは必ずここに紐づく。通知は常に届く。他のプロファイルとは完全分離。

サブプロファイル

  • 追加ユーザーとして新規セットアップ。アプリやデータは完全独立。通知はメインと干渉せず、開いたときのみ反応。

メリット

ストレージ暗号化はプロファイル単位で強固。各ユーザーごとに暗号鍵が分かれているため、メインを開いた状態ではサブのデータにアクセスできない（逆も同じ)。よって、サブでアプリが「汚染」されても、メインには一切影響がない。

ディメリット

  • 写真や動画の保存場所が異なる。メインで撮った写真はサブからは見えない。共有したい場合はクラウドやUSB経由で転送する必要あり。
  • 通知が停止。プロファイルを切り替えている間は、他方の通知は止まります（電話やSMSはメインに届くが、サブ起動時に通知はこず、確認はメインに戻ってから）。切り替え時は画面が一瞬ブラックアウトし、PIN入力後に切り替わる仕組みです。
  • サブを閉じると、そのプロファイルのアプリは完全に停止（音楽再生やダウンロードも継続しません）。メインとサブを「同時に動かす」ことはできない仕様です。

作成方法

設定 → システム → 複数のユーザー → ユーザーを追加。名前をつける（例：「ゲーム用」「実験用」）。「今すぐ切り替え」を選ぶと、初期セットアップが始まる。

実用例

  • メイン：仕事・連絡系アプリのみ → 使用容量は10〜20GB程度に抑える
  • サブ：ゲームや動画アプリを入れる → 容量を多く消費してもメインが軽快に保てる

切り替え方法

通知バー右上のユーザーアイコンをタップ → プロファイルを選択 → パスコード入力 → 切り替わり完了。別のスマホを開いたような感覚で利用できます。

※補足：独立した環境でも、物理的なストレージは共有。例えば128GBモデルのPixelなら、メインとサブ合わせて合計128GBを使います。

Google Playストアをどう使うか？

GrapheneOSではAndroidに導入されているGoogleプレイストアが導入されていない。Sandboxed Google Playという仕組みでPlayストアを利用可能です。

Sandboxed Google Play（Googleプレイストア）のメリットとディメリット

  • メリット：普通のAndroidと同じようにアプリを入れられる／銀行やゲームの互換性が上がる。
  • デメリット：Google関連のプロセスが動くため、若干のプライバシー妥協がある。

よって、実用的な使い方は「メインはGoogleなし」「サブにPlayストアを導入」。メインは純粋に安全・軽量に、サブはGoogle前提のアプリ専用。メインはAurora Storeでアプリをインストールする形が理想的です。

ここでAurora Storeとは

このストアはGoogleアカウント不要でPlayストアアプリをダウンロードできる代替ストア。自動更新は不可（数日に一度チェック必要）。「Googleにログインしたくない」人向け。セキュリティ的にはPlayストアと同じAPKを取得。

実用例

  • 通知必須アプリ（LINE、Gmailなど）はPlayストア経由で更新。
  • 趣味や検証用アプリはAuroraで入れてサブに隔離するのが実用的。

通知の運用ポイント

  • メインに置くべき：電話・SMS・LINE・会社メール（通知必須系）
  • サブに隔離すべき：SNS（X/Instagram/TikTok）、ゲーム、銀行アプリ、怪しい無料アプリ

イメージとしては通知が遅れて困るアプリはメインへ。サブは「後で確認すればいい」アプリ専用に。

Sandboxed Google Playの導入方法と更新について

  • 標準搭載アプリのApp Storeに「Google Play services」と「Google Play Store」をインストール。※片方インストールすればもう一方も自動的にインストールされる。
  • Sandboxed Google Playは通常のAndroidと同じくアプリを自動更新。ただしシステム権限は持たないため、GrapheneOSのセキュリティ基盤は維持されます。

Aurora Storeの導入方法と更新について

  • 標準搭載ブラウザVanadiumで https://auroraoss.com/files/AuroraStore/Release を開く。
  • 「Created at」で一番新しい.apkをダウンロードしてインストール。
  • Aurora StoreはGoogle Playから正規APKを取得するため安全性は高いが、更新が遅れる場合があるので定期的に手動チェック。
  • ダウンロードした古いAPKはストレージ節約のため削除可。

※Aurora Storeには「匿名」「Googleアカウントログイン」の2モードがありますが、基本は匿名で十分。アプリ側でGoogleログインするのはOKだが、Aurora Store本体でGoogleログインするのは非推奨。

日本語入力を有効にする（Gboard導入）

GrapheneOSには標準で日本語入力がありません。そのためSandboxed Google PlayやAurora Storeから「Gboard（Google 日本語入力）」をインストールし、以下の手順で有効化します。

  1. 設定 (Settings) を開く → システム (System) → 言語と入力 (Languages & input)
  2. オンスクリーンキーボード (On-screen keyboard) → キーボードの管理 (Manage keyboards)
  3. Google 日本語入力 (Gboard) をオンにする
  4. 文字入力時に地球マークを長押し → 「Google 日本語入力」を選択

※Gboard以外にも、OpenBoardやMozcなどオープンソース系IMEも利用可能ですが、初心者にはGboardが安定です。

セキュリティソフトとVPN

セキュリティソフト（例：ESET）

  • メイン：GrapheneOS標準で十分だが、入れるメリットもある（フィッシングサイトのブロックや迷惑SMS検知など）。ただしバッテリー消費・重さ増がデメリット。
  • サブ：銀行・課金・ゲーム環境に導入は理にかなう。特にPlayサービス環境では追加の保護が働きやすい。
  • Vanadiumは対象外で、保護機能はChromeなどサポートブラウザでのみ有効。
  • 結論：どうしても導入したいなら「サブだけ」に入れるのがベスト。

VPN

公共Wi-Fi利用が多いなら導入推奨。プロファイルごとに別々に動作可能。VPNはすべての通信を暗号化できるため、公共Wi-Fiや出先での盗聴対策に直結。これはセキュリティソフトでは代替できない領域。 よって、優先度はVPN ＞ セキュリティソフト（GrapheneOSでやるべき基本設定）。

まとめ

VPNは実用的、セキュリティソフトはサブでのみ意味あり。こう運用すれば「セキュリティ最強」と「Google必須アプリの利便性」の両立が可能です。

その他・補足

バックアップ運用

GrapheneOSではGoogleバックアップが完全には機能しない部分があります。代替手段としては Nextcloud / Syncthing / オフラインバックアップ などを利用。

注意点：サブプロファイルを削除すると、そのプロファイルのアプリ・データは丸ごと消去。必要なデータは必ず事前にバックアップ。

バッテリーとパフォーマンス

プロファイルを複数作るとストレージ消費だけでなくメモリも増加。ただし「閉じたサブ」は完全停止するため同時負荷はかからない。電池もちを伸ばすには「通知不要アプリをサブに隔離」「不要な常駐サービスは削除」が有効。

位置情報やセンサー制御

GrapheneOSの強みは権限制御の細かさ。ネットワーク権限拒否 → オフラインで動くアプリからの情報漏洩を防止。センサーオフ（クイック設定）→ 加速度計やジャイロなども無効化。位置情報は「常時許可」ではなく使用中のみが基本。

Pixelに戻す方法

注意

この操作で Pixel内のデータは完全消去 されます。必ずバックアップを取ってから作業してください。コマンドやファイルを間違えると文鎮化（起動不可）のリスクがあります。
公式ROMを書き込んだ後は、fastboot flashing lock を実行してブートローダーを再ロックすれば、セキュリティも純正状態に戻ります。

戻す方法（簡略版）

  1. PCで Googleの公式ファクトリーイメージをダウンロード。
  2. Pixelをブートローダー（Fastboot）モードにして接続。
  3. Windowsなら flash-all.bat を実行（macOS/Linuxは flash-all.sh）。→ 純正Android（Pixel純正ROM）が入る。
  4. 起動後、Googleアカウントでログイン → 連絡先・履歴を同期。

※ データは全部消えるので、バックアップを取ってから進めてください。

必要なものを準備

  • USBケーブル（PixelとPCを接続）
  • パソコン
  • Pixel端末（GrapheneOSが入っていてもOK）
  • Android Platform Tools（ADB/Fastboot）
  • Pixel用ファクトリーイメージ（自分のPixel機種とバージョンを選択）

Pixel端末側の準備

  1. 設定 → デバイス情報 → ビルド番号を7回タップ →「開発者向けオプション」有効化。
  2. 設定 → システム → 開発者向けオプション → OEMロック解除 をオン（PIN要求が出る場合あり）。
  3. 電源を切る → 電源＋音量下を同時押しで起動 → ブートローダー（Fastboot）に入る。

パソコン側の準備（ダウンロード＆展開）

  1. Platform-Tools をダウンロード：
    https://developer.android.com/tools/releases/platform-tools?hl=ja
  2. Pixelファクトリーイメージをダウンロード：
    https://developers.google.com/android/images?hl=ja
  3. 両方の .zip を展開し、同じフォルダにまとめる（例：デスクトップ上の同一フォルダ）。

アンロック & フラッシュ手順

  1. USBでPCに接続。
  2. フォルダの空き領域で Shift + 右クリック(5の画像の箇所) →「ターミナルで開く」 を選択。
  3. 認識確認：
    .\fastboot devices
  4. ブートローダーのアンロック（この時点では書き込みは開始されません）：
    .\fastboot flashing unlock
    → 端末に「Unlock bootloader?」が出るので Yes を選択し電源ボタンで決定。
  1. 最後に同フォルダ内の flash-all.bat（macOS/Linuxは flash-all.sh）を実行。ここでROMの書き込みがスタート。PC画面にログが流れるので完了まで待機。
  1. Pixelが自動で再起動 → Fastboot画面に戻る。
  2. Fastboot画面で「Start」に合わせて電源ボタンで確定 → 起動したら初期セットアップ（Googleアカウントでログイン）。
  3. 最後にもう一度Fastbootで起動し、fastboot flashing lock を実行してブートローダーを再ロック。

ケーブルはフラッシュ完了後なら抜いてOK。PC側のウィンドウも閉じて問題ありません。
